# BTCUSDT 量化盯盘控制台 (假突破/跌破策略)

这是一套基于 Node.js + Express.js 开发的专业级量化盯盘与自动交易信号触发服务。系统专为 **高频短线突破（假突破/假跌破）** 策略设计，结合了币安实时的预测资金费率（Premium Index），并通过 Webhook 动态触发交易报文。

## ✨ 核心特性

- ⚡️ **毫秒级实时判断**：每 90 秒轮询一次币安官方底层 `premiumIndex` 接口，零延迟获取真实市场情绪。
- 🛡️ **铁律级策略防御**：
  - **假突破/假跌破状态机**：精准捕捉价格“触碰边界后回落/反弹”的信号。
  - **资金费率方向硬拦截**：假突破（开空）时资金费率必须为正；假跌破（开多）时资金费率必须为负，否则强制拦截并重置状态。
  - **单次触发锁定**：一旦成功触发开仓报文，系统将永久锁定 `isHoldingPosition`，**绝对防止重复开仓和无限重试爆仓**，必须人工干预重置。
  - **极端行情防死锁**：向上突破时自动清空向下状态，防止“天地针”导致双边挂起死锁。
  - **API 脏数据毒化防御**：遇到币安接口异常或返回 `NaN` 时自动抛弃，绝不污染系统历史图表。
- 📊 **极客风数据仪表盘**：
  - 内置基于 ECharts 的精美深色高级图表（含渐变折线、K线影线）。
  - **双图表对比**：8小时官方结算历史（大趋势） + 纯1小时实时预测费率走势（短线敏锐度）。
- ⚙️ **完全动态可配置**：
  - 支持在前端 Web 页面随时填写/修改 **触发价格**、**Webhook 地址**、**JSON 报文**。
  - 报文支持 `{{timenow}}` 动态时间戳变量替换。
  - 修改配置时系统会自动挂起监控，保证配置过程绝对安全。
- 🔔 **飞书报警集成**：所有价格触碰、状态重置、开仓拦截、致命网络错误等都会第一时间推送至飞书。

## 🚀 快速开始

### 1. 环境要求
- Node.js (建议 v16+)
- npm

### 2. 安装与启动
```bash
# 进入项目目录
cd /Users/qihong/demo/pythondingpan

# 安装依赖
npm install

# 启动服务
npm start
```

### 3. 配置与监控
服务启动后，在浏览器中访问：**http://localhost:3000**

1. 在左上角的输入框中填写你的 **阻力价**（假突破开空用）和 **支撑价**（假跌破开多用）。
2. 填写接收开仓信号的 **Webhook URL**。
3. 填写多空触发时的 **JSON 报文**（可根据你的交易中间件自定义字段）。
4. 点击 **【保存配置并锁定】**，此时系统正式进入实盘监控状态。

## 🧠 策略原理详解

### 🔴 做空逻辑 (假突破)
1. 实时价格 **>= 阻力价**：系统挂起“监控回落”状态（向下的监控状态会被强制清空）。
2. 随后价格 **< 阻力价**（产生回落）：
   - 如果此时 **实时预测资金费率 > 0**：触发【做空】Webhook，并永久锁定系统。
   - 如果此时 **实时预测资金费率 <= 0**：拦截开空请求，重置突破状态，等待下一次机会。

### 🟢 做多逻辑 (假跌破)
1. 实时价格 **<= 支撑价**：系统挂起“监控反弹”状态（向上的监控状态会被强制清空）。
2. 随后价格 **> 支撑价**（产生反弹）：
   - 如果此时 **实时预测资金费率 < 0**：触发【做多】Webhook，并永久锁定系统。
   - 如果此时 **实时预测资金费率 >= 0**：拦截开多请求，重置跌破状态，等待下一次机会。

## ⚠️ 重要注意事项

1. **飞书 Webhook 替换**：
   请打开 `server.js`，将顶部的 `FEISHU_WEBHOOK` 变量替换为你真实的飞书机器人地址，否则你将收不到报警信息。
2. **数据源说明**：
   程序右上角的“实时资金费率”读取的是币安预测的下一期费率（Premium Index），它反映了市场当下这一秒的多空博弈情况，与第三方平台（如非小号）展示的“历史已结算费率”可能出现正负背离。**在短线交易中，请绝对相信本系统抓取的实时预测费率。**
3. **解除持仓锁定**：
   触发开仓后系统会自我冻结（为了防止网络抖动导致的无限重复开仓）。你必须登录网页端，点击红色的 **【修改配置 / 重置状态 (解锁)】** 按钮，系统才会重新开始监控。
4. **数据持久化**：
   当前配置保存在 Node.js 服务端的内存中，如果重启了 Node 进程，你需要重新在网页端填写配置。
